# 快速上手指南

本指南帮助你在 5 分钟内开始使用 AI Project Initiator。

## 第一步：选择启动方式

根据你的需求选择：

| 场景 | 推荐方式 | 时间 |
|------|---------|------|
| 快速验证想法 | 最小化预设 | 1-2h |
| 团队项目规划 | 专业预设 | 2-4h |
| 完整产品开发 | 全栈预设 | 3-6h |

## 第二步：准备项目信息

在开始前，准备好以下信息：

**必需**：
- 项目类型（Web/Mobile/API/Desktop）
- 项目名称
- 一句话描述

**可选但推荐**：
- 目标用户群体
- 核心痛点
- 时间和资源约束

## 第三步：启动对话

### 使用最小化预设

```
@project-init

项目类型: Web应用
项目名称: 任务管理工具
一句话描述: 帮助个人用户高效管理日常任务的轻量级工具
```

AI 会引导你完成 5 个阶段：
1. 需求澄清（15 个关键问题）
2. 生成 PRD
3. 定义 MVP
4. 设计技术架构
5. 制定开发路线图

### 使用专业预设

如果你需要更专业的规划：

```
我想启动一个新项目，使用专业预设

项目类型: SaaS平台
项目名称: 客户关系管理系统
一句话描述: 帮助中小企业管理客户关系和销售流程

请使用 @pm-kit 进行功能优先级排序
请使用 @doc-coauthor 协作编写 PRD
```

## 第四步：回答问题

AI 会按顺序提问，你可以：
- 简短回答（AI 会追问细节）
- 详细描述（AI 会总结要点）
- 说"不确定"（AI 会提供选项）

**提示**：不用担心回答不完美，AI 会在每个阶段结束时让你确认和调整。

## 第五步：审阅产出

每个阶段结束后，你会得到：
- Phase 2: `docs/PRD.md`
- Phase 3: `docs/MVP.md`
- Phase 4: `docs/ARCHITECTURE.md` + 项目结构
- Phase 5: `docs/ROADMAP.md`

仔细审阅每个文档，提出修改意见。

## 第六步：开始开发

完成所有阶段后，你将拥有：
- ✅ 清晰的产品需求文档
- ✅ 明确的 MVP 范围
- ✅ 技术架构方案
- ✅ 可执行的开发计划
- ✅ 编辑器配置文件

现在可以开始写代码了！

## 常见问题

**Q: 我可以跳过某些阶段吗？**  
A: 可以。如果你已经有 PRD，可以直接说"跳到技术架构"。

**Q: 我可以中途暂停吗？**  
A: 可以。所有产出都保存在 `docs/` 目录，随时可以继续。

**Q: 我需要安装什么？**  
A: 基础功能不需要安装。如果使用扩展 skills（如 pm-kit、web-test），需要安装 Python 依赖。

**Q: 如何选择技术栈？**  
A: AI 会在 Phase 4 根据你的需求推荐 2-3 个方案，并说明优劣。你可以选择或提出自己的方案。

**Q: 生成的文档可以修改吗？**  
A: 当然。这些文档是起点，你可以随时修改和完善。
